## 2. Corrección de balances en Cuentas Corrientes

### Archivos modificados
- `src/pages/CustomersPage.jsx`

### Bug corregido A — Double counting en `computeAllocations`

**Problema:** `totalBalance` sumaba TODOS los movimientos (incluyendo cargos de pedidos con `saleId`).
Al calcular `initialDebt = max(0, -totalBalance)`, los cargos de pedidos quedaban incluidos en la
"deuda inicial". Pero `getUnpaidOrders` también mostraba esos pedidos por separado → se contaban DOS veces.

```js
// ANTES (línea ~42)
const totalBalance = accountPayments
  .filter(p => p.customerId === customerId)
  .reduce((sum, p) => p.type === "payment" ? sum + p.amount : sum - p.amount, 0);

// DESPUÉS
const totalBalance = accountPayments
  .filter(p => p.customerId === customerId && !p.saleId)  // ← solo movimientos sin saleId
  .reduce((sum, p) => p.type === "payment" ? sum + p.amount : sum - p.amount, 0);
```

### Bug corregido B — Crédito nunca se consumía al aplicarse a pedidos

**Problema:** Al pagar un pedido con crédito, se creaba un `payment` para el pedido pero el crédito
del apertura nunca se "gastaba". Con el tiempo, el crédito parecía multiplicarse.

```js
// ANTES — paso 1 de registerPayment
for (const { sale, creditApplied } of creditForOrders) {
  const p = { ..., type: "payment", ... };
  await supabase.from("account_payments").insert(...);
  allNewPayments.push(p);
  balanceDelta -= creditApplied;  // solo afectaba customers.balance (campo no usado en display)
}

// DESPUÉS — agrega cargo de consumo sin saleId
for (const { sale, creditApplied } of creditForOrders) {
  const p = { ..., saleId: sale.id, type: "payment", paymentMethod: "balance", notes: "Saldo a favor aplicado" };
  await supabase.from("account_payments").insert(...);
  allNewPayments.push(p);

  const consumption = {
    id: crypto.randomUUID(), customerId: payModal.id, saleId: null,
    amount: creditApplied, type: "charge", paymentMethod: "balance",
    date: todayStr(), notes: "Crédito consumido",
  };
  await supabase.from("account_payments").insert(accountPaymentToDb(consumption));
  allNewPayments.push(consumption);
  // balanceDelta ya no se modifica (efecto neto en custBal = 0)
}
```

**Para revertir:** restaurar el filtro sin `&& !p.saleId` y quitar el bloque de `consumption`.
